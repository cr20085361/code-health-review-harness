# Security Policy

## 支持范围

当前维护分支：`main`

当前版本：`0.1.x`

## 安全边界

本项目的默认运行原则是：

- 允许读取代码、配置、文档和测试输出。
- 允许运行 build/test/lint/audit 等验证命令。
- 禁止删除、重置、部署、迁移、自动修复、上传代码、读取 secrets 或输出敏感信息。
- 禁止在未审查和未确认的情况下安装外部 skill、包或二进制文件。

## 报告安全问题

如果发现脚本可能执行破坏性命令、泄露敏感信息、越权读取凭据，或 skill 指令可能诱导 agent 忽略安全边界，请提交 security issue 或私下联系仓库维护者。

报告时请包含：

- 受影响文件。
- 触发步骤。
- 预期安全边界。
- 实际行为。
- 建议修复方向。

## 禁止行为

不要在 issue、PR 或报告样例中粘贴真实 token、密码、cookie、私钥、数据库连接串或客户数据。
