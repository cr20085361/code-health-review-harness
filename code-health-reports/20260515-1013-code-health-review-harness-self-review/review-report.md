## Findings

### CHR-20260515-001 - collect-repo-facts absolute OutputPath fails

- Severity: Medium
- Status: verified-closed
- Evidence: skills/code-health-review-harness/scripts/collect-repo-facts.ps1 line 172; command reproduction failed with NotSupportedException when OutputPath was absolute.
- Impact: Users can hit path-format failures when trying to save facts to an absolute artifact path.
- Recommendation: Reuse the Resolve-InputPath helper pattern from summarize-review-history.ps1.
- Verification: VER-20260515-001.
- Confidence: High.

### CHR-20260515-002 - run-safe-checks does not discover PowerShell project validation scripts

- Severity: Medium
- Status: verified-closed
- Evidence: artifacts/safe-checks.json returned an empty results array, while scripts/verify-project.ps1 and scripts/test-harness-smoke.ps1 exist and pass.
- Impact: Reviews of PowerShell/documentation projects can show no safe checks despite available project-defined validation.
- Recommendation: Add conservative PowerShell validation discovery or a repository allowlist/config mechanism.
- Verification: VER-20260515-002.
- Confidence: High.

### CHR-20260515-003 - install-user-skill WhatIf prints a misleading success message

- Severity: Medium
- Status: verified-closed
- Evidence: scripts/install-user-skill.ps1 line 60 prints Installed skill outside the ShouldProcess branch; -WhatIf command printed that message after simulated operations.
- Impact: A dry run can be mistaken for a completed global skill sync.
- Recommendation: Print install success only after the copy branch executes; print a distinct dry-run message for WhatIf.
- Verification: VER-20260515-003.
- Confidence: High.

### CHR-20260515-004 - usage guide has a duplicated notice before the title

- Severity: Low
- Status: verified-closed
- Evidence: docs/usage.md line 1 and line 93 contain the same code-health-reports notice.
- Impact: The usage guide starts with an orphan note before its H1 and looks less polished.
- Recommendation: Remove the line-1 duplicate and keep the contextual note near closure artifacts.
- Verification: VER-20260515-004.
- Confidence: High.

### CHR-20260515-005 - smoke test leaves an empty .tmp parent directory

- Severity: Low
- Status: verified-closed
- Evidence: After scripts/test-harness-smoke.ps1 passed, .tmp existed and was empty; cleanup removes .tmp children but not the parent.
- Impact: Local smoke runs leave ignored residue and slightly weaken cleanup expectations.
- Recommendation: Remove .tmp when empty after cleanup, or document it as intentional cache state.
- Verification: VER-20260515-005.
- Confidence: High.
